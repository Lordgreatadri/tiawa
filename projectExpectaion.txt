🔷 6. Technical Documentation (API Overview)
Base URL
https://api.tiawaa.com/v1
________________________________________
🔗 Create Short URL
POST /links
Request:
{
  "long_url": "https://example.com/very-long-url",
  "custom_alias": "optional",
  "generate_qr": true
}
Response:
{
  "short_url": "https://tiawaa.com/abc123",
  "qr_code_url": "https://tiawaa.com/qr/abc123.png",
  "id": "link_id"
}
________________________________________
📊 Get Analytics
GET /links/{id}/analytics
Response:
{
  "clicks": 1200,
  "devices": {
    "mobile": 800,
    "desktop": 300
  },
  "locations": [
    {"country": "Ghana", "count": 900},
    {"country": "Nigeria", "count": 200}
  ]
}

🔷 2. API Authentication & Security
This part matters more than the UI long-term.
________________________________________
🔐 Authentication Strategy
Option A: JWT (Recommended)
Flow:
1.  User logs in:
POST /auth/login
Response:
{
  "access_token": "JWT_TOKEN",
  "expires_in": 3600
}
2.  Frontend stores token:
• Memory (best) OR
• HTTP-only cookie (more secure)
3.  Every request:
Authorization: Bearer JWT_TOKEN
________________________________________
🔑 API Keys (For Developers)
Separate from user login.
• Each user gets API key:
x-api-key: tiawaa_xxxxx
Used for:
• Programmatic link creation
• Analytics access
________________________________________
🚧 Rate Limiting
You NEED this if you offer free tier.
Strategy:
Per IP (unauthenticated users)
• 10 links/hour
• 100 redirects/minute
Per user (authenticated)
• Based on plan:
o Free: 100/day
o Paid: scalable
Per API key
• 60 requests/minute
________________________________________
🧠 Abuse Prevention
• CAPTCHA on free tier link creation
• Block:
o Spam domains
o Phishing URLs
• Optional:
o URL safety check (Google Safe Browsing API)

🔷 3. Laravel Blade vs SPA (Important Decision)
Short answer:
👉 Yes, Laravel Blade can work—but it’s not ideal for your long-term vision.
Let’s break it honestly.
________________________________________
⚖️ Option 1: Laravel Blade (Server-rendered UI)
✅ Pros:
• Fast to build (especially if you know Laravel)
• Built-in auth system
• Clean structure
❌ Cons:
• Not great for heavy API-driven apps
• Harder to scale frontend independently
• Less “modern SaaS feel”
________________________________________
⚖️ Option 2: API + Frontend (Recommended)
Stack:
• HTML + Tailwind
• Alpine.js OR Vue/React
✅ Pros:
• Clean separation (frontend ↔ API)
• Easier to scale
• Works great with mobile apps later
• More interactive UX
❌ Cons:
• Slightly more setup
________________________________________
💡 Best Hybrid Approach (Strong Recommendation)
Use:
• Laravel → Backend API ONLY
• Frontend → Blade + Alpine.js (initially)
Why?
• You get speed + flexibility
• Can later upgrade to full SPA
________________________________________
🔧 Suggested Stack
• Tailwind CSS → UI
• Alpine.js → Interactivity
• Axios / Fetch → API calls
